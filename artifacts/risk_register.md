# Список Рисков

## 1. Workspace быстро захламится

Последствие:

- команда перестанет ему доверять.

Mitigation:

- держать простую структуру;
- регулярно архивировать нерелевантное;
- обновлять только ключевые trackers.

## 2. AI начнет производить красивый шум

Последствие:

- решения будут приниматься на слабой фактуре.

Mitigation:

- требовать evidence и ссылки;
- разделять факт, гипотезу и интерпретацию;
- обновлять артефакты только после проверки.

## 3. Несколько пользователей начнут ломать общие файлы

Последствие:

- потеря актуальности и конфликтующие изменения.

Mitigation:

- закрепить owner для контуров;
- вести git как source of truth;
- использовать chat-generated правки осмысленно, а не хаотично.

## 4. Dashboard станет слишком сложным

Последствие:

- команда перестанет им пользоваться каждый день.

Mitigation:

- держать основной dashboard простым;
- выносить частные use case в generated dashboards;
- не смешивать все запросы в один UI.

## 5. Контакты и материалы будут храниться без follow-up

Последствие:

- knowledge base станет пассивным архивом.

Mitigation:

- у каждой записи должны быть owner и next step;
- задачи должны обновляться вместе с контактами и research library.

## Что нужно сделать дальше

- ранжировать риски по вероятности и влиянию;
- назначить owner на top-5 рисков;
- раз в неделю пересматривать риск-регистр вместе с backlog.
