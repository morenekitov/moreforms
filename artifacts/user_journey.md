# User Journey

## Journey совместной discovery-команды

### 1. Появляется новая идея или сигнал

Источник:

- разговор;
- интервью;
- статья;
- видео;
- конкурент;
- наблюдение из рынка.

### 2. Идея фиксируется

Команда добавляет запись в `ideas.csv`:

- название;
- thesis;
- сегмент;
- статус;
- evidence;
- next step.

### 3. Сбор контекста

Добавляются:

- материалы в `research_library.csv`;
- конкуренты;
- контакты;
- задачи на интервью и анализ.

### 4. Синтез

Команда:

- обновляет one-pager;
- уточняет JTBD;
- связывает находки с текущей thesis;
- решает, нужен ли prototype или generated dashboard.

### 5. Прототип / dashboard

Если нужно быстрое представление:

- создается markdown dashboard;
- или дорабатывается основной Streamlit UI.

### 6. Решение

Команда выбирает:

- углублять идею;
- сузить сегмент;
- сменить thesis;
- остановить направление.

## Что нужно валидировать

- сколько шагов реально проходит каждая идея;
- где команда теряет скорость;
- в какой момент уже нужен prototype, а где достаточно evidence;
- какие generated dashboards реально помогают принимать решения.
