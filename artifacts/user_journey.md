# User Journey

## Journey первой итерации

### 1. Возникает задача

Пользователю нужно:

- собрать форму;
- провести опрос;
- обработать Excel;
- подготовить отчет по результатам.

### 2. Формулировка запроса

Пользователь описывает задачу текстом или через видео-бриф.

Ключевой вопрос:

- достаточно ли такой формулировки, чтобы автоматически собрать рабочую форму.

### 3. Генерация формы

Система предлагает Яндекс.Форму:

- с полями;
- логикой вопросов;
- базовой структурой сценария.

Ключевой риск:

- форма потребует слишком много ручной доработки.

### 4. Сбор ответов

Ответы приходят в Яндекс.Формы, затем результаты загружаются в систему.

Ключевой риск:

- пользователь не понимает, где начинается аналитика и как перенести данные без боли.

### 5. Анализ по промпту

Пользователь задает вопрос к данным.

Система строит:

- summary;
- графики;
- шаблоны визуализаций;
- предлагаемые выводы.

Ключевой риск:

- анализ выглядит красиво, но недостаточно полезен для принятия решений.

### 6. Интерпретация и экспорт

Пользователь проверяет выводы, дополняет их и выгружает результат.

Ключевой риск:

- результат сложно использовать дальше в привычной коммуникации.

## Что нужно валидировать

- где больше всего времени теряется сейчас;
- где пользователь хочет автоматизации, а где контроля;
- в какой точке чаще всего возникает недоверие к системе.
