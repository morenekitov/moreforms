# One-Pager

## Проблема

У команды discovery быстро накапливаются:

- идеи;
- заметки;
- ссылки;
- конкуренты;
- интервью;
- видео и статьи;
- куски прототипов;
- backlog гипотез;
- задачи и follow-up.

Если это разложено по чатам, заметкам и личным файлам, команда теряет контекст и медленно принимает решения.

## Ценность

`moreforms` дает один AI-first workspace для совместного venture discovery:

`idea -> evidence -> competitor map -> interviews -> prototype -> decision`

Система помогает не только хранить знания, но и быстро превращать их в views, generated dashboards и рабочие next steps.

## Для кого

- фаундеры;
- product / strategy lead;
- исследователи;
- технический co-builder;
- операционные участники discovery-процесса;
- внешние advisors, если им нужен read-only контур.

## Какой результат получает команда

- общая база идей и материалов;
- единая версия текущей thesis;
- прозрачный backlog;
- понятный контактный pipeline;
- быстрые generated dashboards под конкретные вопросы;
- меньше потерь контекста между людьми и итерациями.

## Почему сейчас

- скорость learning loops стала критичнее количества встреч;
- AI позволяет быстро структурировать слабые сигналы и превращать их в рабочие документы;
- чем раньше команда начнет вести evidence централизованно, тем меньше хаоса будет при росте discovery.

## In Scope текущей версии

- shared trackers для идей, задач, контактов и материалов;
- discovery-артефакты;
- competitor intelligence;
- chat dashboard на OpenClaw;
- lightweight generated dashboards.

## Out of Scope текущей версии

- полноценная CRM;
- data warehouse и BI platform;
- тяжелая проектная система;
- отдельный multi-tenant продукт для внешних клиентов.

## Что нужно валидировать

- действительно ли команде удобнее работать через единый workspace;
- какие 3-5 views используются чаще всего;
- где полезнее markdown dashboards, а где нужен полноценный runtime UI;
- какие знания должны обновляться автоматически, а какие вручную.
