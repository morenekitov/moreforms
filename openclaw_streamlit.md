# OpenClaw x Streamlit

## Текущая схема интерфейсов

В проекте `moreforms` есть два пользовательских URL-режима в одном Streamlit-приложении:

- основной дашборд: `https://app.moreforms.ru`
- чат-дашборд: `https://app.moreforms.ru?view=chat`

Основной дашборд нужен для:

- конкурентной карты;
- внедренческих сигналов;
- продуктовых артефактов;
- просмотра созданных generated dashboards.

Чат-дашборд нужен только для работы с OpenClaw.

## Как OpenClaw используется в Streamlit

Streamlit не ходит напрямую в сложный OpenClaw HTTP gateway-контур.

Вместо этого используется внутренний HTTP-adapter, который:

- принимает `POST /v1/responses`;
- вызывает `openclaw agent --json`;
- возвращает `output_text` в формате, который уже понимает чат в Streamlit.

Это нужно, чтобы упростить серверную интеграцию и избежать лишней device/operator auth-логики в UI.

## Правила работы с дашбордами

Если пользователь просит:

- создать новый дашборд;
- доработать старый дашборд;
- собрать отдельную страницу под use case;

то действуй так.

### 1. Lightweight dashboards

Если достаточно markdown/dashboard-page без отдельной сложной логики:

- создай или обнови файл в `generated_dashboards/<slug>.md`
- `slug` должен быть коротким, в lowercase, через дефисы или `_`

После этого верни ссылку:

- `https://app.moreforms.ru?dashboard=<slug>`

### 2. Доработка основного дашборда

Если нужно менять:

- общую структуру основного дашборда;
- табы;
- фильтры;
- таблицы;
- карточки;
- query-param routing;

то редактируй:

- `app.py`

### 3. Доработка чат-дашборда

Если нужно менять:

- режимы чата;
- быстрые действия;
- логику prompt building;
- вывод ответа;
- инструкции для OpenClaw;

то редактируй:

- `app.py`
- `openclaw_agent.md`
- `openclaw_streamlit.md`
- `deploy/openclaw_responses_adapter.py` при необходимости

## Что отвечать пользователю после создания дашборда

Если дашборд создан или обновлен, ответ должен содержать:

- короткий вывод;
- какие файлы созданы или изменены;
- прямую ссылку на дашборд.

Пример:

- `Ссылка: https://app.moreforms.ru?dashboard=university-intake-summary`

## Ограничения

- не создавай новый Python entrypoint без необходимости, если задачу можно решить через `generated_dashboards/<slug>.md`
- не ломай основной дашборд ради одного кастомного use case
- не придумывай ссылку без реального `slug` и созданного файла
