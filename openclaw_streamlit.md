# OpenClaw x Streamlit

## Поверхности

В проекте есть:

- основной dashboard с двумя top-level табами: `Чат` и `Пространство`
- отдельный URL чат-дашборда: `https://app.moreforms.ru?view=chat`

## Основной dashboard

Основной dashboard показывает только рабочие сценарии команды:

- конкуренты;
- сигналы внедрений;
- контакты;
- артефакты;
- проведенные интервью;
- бэклог и требования.

## Чат-дашборд

Top-level таб `Чат` и отдельный чат-дашборд содержат только:

- выбор режима;
- статус backend;
- сам чат.

Таблицы created dashboards в чате быть не должно.

## Пространство

`Пространство` содержит подтабы:

- конкуренты;
- сигналы внедрений;
- контакты;
- артефакты;
- интервью;
- бэклог и требования;
- созданные дашборды.

## Правила изменений

### Если нужно обновить tracker

Обновляй соответствующий CSV:

- `data/competitors.csv`
- `data/adoption_mentions.csv`
- `data/contacts.csv`
- `data/interviews.csv`
- `data/backlog.csv`
- `data/artifacts.csv`

### Если нужно обновить продуктовую логику

Обновляй:

- `artifacts.md`
- `artifacts/*.md`

### Если нужно создать lightweight dashboard

- создай или обнови `generated_dashboards/<slug>.md`
- верни ссылку вида `https://app.moreforms.ru?dashboard=<slug>`

### Если нужно поменять Streamlit UI

Меняй:

- `app.py`

## Ограничения

- не добавляй лишние разделы в основной dashboard без прямого запроса;
- не превращай chat dashboard в второй основной dashboard;
- не придумывай ссылку без реально созданного slug.
